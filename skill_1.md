This skill is used to **analyze and redesign existing websites** for accounting, financial, or professional services.

The goal is NOT to create flashy or experimental designs, but to:
- Increase **trust**
- Improve **conversion**
- Add **distinctive but controlled visual identity**

---

## CORE PRINCIPLE

Balance:

- 70% — Trust & professionalism  
- 20% — Structure & UX clarity  
- 10% — Creative differentiation  

DO NOT break this balance.

---

## CONTEXT UNDERSTANDING

Before making any design decisions, analyze:

### 1. Business type
- Accountant, tax consultant, financial service
- Target: small business / entrepreneurs / marketplace sellers

### 2. User intent
Users come to:
- Avoid risks (taxes, fines)
- Gain control
- Delegate responsibility

Design must reduce anxiety and increase confidence.

---

## REDESIGN GOALS

Every redesign must improve:

- Perceived reliability
- Clarity of services
- Ease of decision-making
- Professional positioning

---

## WHAT TO AVOID (CRITICAL)

Never use:

- Overly playful or “startup-like” visuals
- Bright/neon/acid colors
- Trendy but meaningless UI elements
- Complex or chaotic layouts
- Generic AI-style templates
- Overuse of animations

This destroys trust.

---

## VISUAL STRATEGY

### 1. Style direction (choose ONE and stay consistent)

- Fintech control interface (dashboard-like)
- Minimal legal / corporate style
- Light fintech modern clean

Do NOT mix multiple styles.

---

### 2. Color system

Use controlled palette:

- Base: white / light gray / dark navy
- Primary: deep blue
- Accent: muted green (optional)

Avoid:
- neon
- excessive gradients
- random colors

---

### 3. Typography

Priority: readability and authority

Allowed:
- Clean sans-serif (Inter, IBM Plex Sans, etc.)
- Optional accent serif for headings

Avoid:
- decorative fonts
- playful fonts
- overly experimental typography

---

### 4. Layout

Use:

- Clear hierarchy
- Structured blocks
- Grid alignment
- Generous spacing

Enhancements allowed:
- Slight asymmetry
- Layering
- Card-based UI

Avoid:
- chaotic layouts
- broken grids
- excessive overlapping

---

### 5. UI Elements

Prefer:

- Cards
- Status indicators
- Checklists
- Step-by-step blocks
- Data-like presentation

This reinforces “control” perception.

---

### 6. Motion

Use minimal motion:

- Soft fade-in
- Hover feedback
- Micro-interactions

Avoid:
- large animations
- distracting transitions
- “wow-effects”

---

## TRUST LAYER (MANDATORY)

Every redesign MUST include:

- Clear offer (what problem is solved)
- Proof (cases, numbers, results)
- Guarantees
- Process transparency
- Contact clarity

If these are missing → redesign is incomplete.

---

## DIFFERENTIATION (IMPORTANT)

Do NOT differentiate through decoration.

Differentiate through:

- Positioning (e.g. “financial control system”)
- Message clarity
- UX logic
- Visual consistency

---

## REDESIGN PROCESS

Follow this sequence:

### Step 1 — Audit
- What looks generic?
- What breaks trust?
- What confuses user?

### Step 2 — Simplify
- Remove visual noise
- Improve structure
- Clarify hierarchy

### Step 3 — Strengthen trust
- Add proof elements
- Improve wording blocks
- Highlight key benefits

### Step 4 — Apply visual system
- Colors
- Typography
- UI consistency

### Step 5 — Add controlled uniqueness
- One distinctive element:
  - layout
  - visual motif
  - UI style

Only ONE. Not multiple.

---

## OUTPUT FORMAT

When responding, always provide:

1. Audit of current design
2. Key problems
3. Redesign concept (clear direction)
4. Specific UI recommendations
5. Visual style description
6. If needed — updated code (HTML/CSS/JS)

---

## FINAL RULE

This is NOT a creative playground.

This is a **trust-building financial interface**.

Design must feel:

- Reliable
- Clear
- Controlled
- Professional
- Modern (but not trendy)

If it looks “cool” but not “trustworthy” → it is wrong.
